4.10 Error Viewer Applet

Functionality to display the following error messages is available.

  • Error messages based on the result codes returned by the SDK APIs
  • Application-specific error messages

Error Display Specifications

This section describes the primary specifications for displaying errors using the error viewer applet.

Display Patterns

Two layouts are available for the error viewer applet: dialog box display and full-screen display.
Each error can be displayed with any of the following patterns, using combinations of these layouts.

Dialog Box Display Only

This pattern is completed with a simple dialog box display.

Figure 4-2 Error Displayed Using Only a Dialog Box

Dialog Box and Full-Screen Display

This pattern is for more complex errors that require a detailed description that do not fit within a dialog box.

The dialog box appears first. Clicking a specific button within that dialog box switches to full-screen display.
The gist of the error appears in the dialog box. A detailed description appears in the full-screen display.

Figure 4-3 Error Displayed Using Both a Dialog Box and Full-Screen Display

Full-Screen Display Only

In general, this option is not used.

Figure 4-4 Error Displayed Using Only Full-Screen Display

Elements to Display

The following elements can be displayed in both the dialog box and full-screen displays.

Note

The error code determines the elements and content that are displayed. An error code list with a provisional name is being prepared.

Error Codes

  • Each error is assigned an ID number. Customer support needs this ID.
  • An error code always appears when displaying errors using the error viewer applet.

Error Message

  • Make sure that the error text guides the user in resolving the error by describing the error's cause, the current status, and any workarounds.
  • When an error uses both dialog box and full-screen display, separate error messages can be displayed for each.

Buttons

  • Up to three buttons can be displayed.
  • When an error uses both dialog box and full-screen display, separate buttons can be set for each.
  • The button on the left responds when the controller's B Button (or a button with equivalent functionality) is pressed, when there are at least two buttons visible.

Determining a Display Method

Display methods are determined based on whether you use dialog boxes or full-screen display for error messages.

Application-Specific Errors

There is functionality for displaying a desired error message together with an application-specific error code.
Use as necessary.

Advantages of Using This Functionality

  • You can use error codes that do not appear in other applications.
    You can use this when you want to associate error codes with workarounds on a website.
  • There are instances when the development load for implementing an error display user interface can be decreased.

Elements Specified From Applications

When displaying application-specific errors, specify the following elements and call the error viewer.

Last Four Error Code Digits

Application-specific errors use error codes in the following format: 2-XXXXX-0000.
Specify the four digits that replace the 0000 portion when calling the error viewer.

Note: The XXXXX portion is automatically set by the application code.

Error Message

For each error, you can specify an error message for dialog boxes, and an error message for the full-screen display.
Display patterns are determined using the same standards described in Determining a Display Method.

Button Language

You can specify the language for the message appearing within buttons.
Use the same language as for the error message passed to the error viewer.

The error viewer will automatically set the messages that appear in buttons.
You cannot specify strings.
