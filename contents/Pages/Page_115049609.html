4.9 Offline Web Applet

The Offline Web Applet is for displaying offline HTML within applications.

Features

  • It can be used to display descriptions of gameplay or other notes within the application.
  • It cannot display server-based webpages.
  • Video playback is possible.
  • The basic specifications are nearly identical to those of the web applet. For more information about the web applet specifications, see 4.8 Web Applet.
Note

The electronic manual that was required for Wii U and Nintendo 3DS has been discontinued. Developers are free to choose whether to include materials within their applications, like gameplay guides that were included in the electronic manual. You can use the Offline Web Applet to display materials like a gameplay guide, but there is no set location, like in the System Menu, to display such content.


Web Applet Differences

Note the following differences between the offline web applet and the web applet during development.

Format Restrictions

  • Unlike EPUB files, there is no metadata for offline HTML.
  • Offline HTML cannot include any of the following web server content: images, video, JavaScript, and CSS.
  • There is no support for cookies or localStorage. (sessionStorage is available.)

Web Fonts

To enable you to reference web fonts from HTML or CSS, store them in the offline HTML folder.

Important Information Specific to Setting Paths

You can use relative paths. You cannot, however, specify a level above the root folder.
Do not use the abbreviated denotations often seen online, such as index.html or index.php, in the offline web applet.
If you do so, an error page indicating that the content does not exist will appear.

Conditions for Pages to Display

Make sure that content to be displayed using offline HTML conforms to the NX guidelines.
